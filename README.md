# Competitive Programming Notebook

This repository contains my personal collection of solutions to various competitive programming problems. It serves as a centralized notebook for tracking my progress, storing implementations of classic algorithms, and maintaining a library of solved problems across different platforms.

## Directory Structure

The repository is organized primarily by platform and topic:

- **atcoder/**: Solutions to problems from AtCoder contests.
- **codeforces/**: Solutions to Codeforces rounds and problemset entries.
- **cses/**: Solutions to the CSES Problem Set.
- **uva/** / **spoj/** / **usaco/**: Platform-specific directories for problems from Online Judge, SPOJ, and USACO.
- **icpc/**: Materials and problems related to ICPC regions and training.
- **clasicos/**: Implementations of standard algorithms and data structures (DP, Graphs, Number Theory, etc.).
- **templates/**: Reusable code templates and cheatsheets.
- **icpc-training-os/**: Tools and scripts for tracking training statistics.

## Purpose

The main goal of this notebook is to:
1. Maintain a history of solved problems for future reference.
2. Build a reliable library of algorithms and data structures.
3. Track training consistency and performance.

## Usage

Most solutions are implemented in C++, following standard competitive programming conventions. Each file is typically named after the problem ID or title.
